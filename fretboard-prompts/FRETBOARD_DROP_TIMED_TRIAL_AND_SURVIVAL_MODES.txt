Root problem:
The current standard game combines a 60-second timer with three lives. This creates an unclear hybrid rule set. A player should immediately understand whether the goal is to score as many notes as possible before time expires or survive until three missed picks end the run.

This is also a marketing problem. Two simple, clearly named modes are easier to explain, demonstrate, and promote:
- Timed Trial: beat the clock and set a high score.
- Survival: protect three picks and last as long as possible.

Engineering review:
Compare these options before changing code:

1. Small patch to current setup
Keep the hybrid mode and only add explanatory copy.
- Lowest complexity
- Does not solve the confusing rule structure
- Poor long-term product clarity

2. Recommended focused refactor
Add an explicit run-mode selection and derive timer/lives/end conditions from that mode while reusing the current reducer, target timing, renderers, scoring, persistence patterns, and results screen.
- Moderate, contained change
- Low maintenance burden
- Solves the gameplay and marketing problem
- No new packages, backend, database, or effects system

3. Larger game-mode architecture
Create a generalized plugin or strategy framework for future modes.
- More abstraction and testing
- Higher maintenance burden
- Overkill for the current stage

Choose Option 2 unless repository inspection reveals a smaller equally clear implementation.

GOAL

Offer two distinct normal-play modes:

TIMED TRIAL
- Fixed 60-second run.
- The run ends only when the timer reaches zero.
- Do not display or use lives.
- A missed target records a miss, resets combo, shows the existing miss reveal/cue, then continues with a new target.
- Wrong fret or wrong string does not remove time or end the run.
- Primary goal is highest score within 60 seconds.
- Results should emphasize score, accuracy, wrong taps, misses, and best streak.

SURVIVAL
- Start with three lives represented by the existing life-pick icons.
- No fixed 60-second countdown.
- The run ends when the third unanswered target reaches the Play Gate.
- Wrong fret or wrong string does not remove a life.
- Correct notes increase score and combo.
- The current pacing progression may remain unchanged in this prompt.
- Results should emphasize notes found, survival time, accuracy, wrong taps, misses, and best streak.

GAMEPLAY RULES

Timed Trial:
- Keep DROP_RUN_DURATION_MS at 60 seconds.
- Ignore lives as a completion condition.
- On target timeout, record the miss and show existing feedback.
- After the miss reveal/recovery completes, spawn the next target.
- Continue until the timer reaches zero.

Survival:
- Do not decrement a hidden timer.
- Track elapsed run time from the existing performance.now-based runStartedAt clock.
- On target timeout, decrement lives.
- Complete the run only when lives reach zero.
- Continue spawning targets indefinitely until the third miss.
- Ensure there is no artificial 60-second cutoff.

For both modes:
- Keep target-specific startedAt and durationMs timing intact.
- Keep the current performance.now clock domain.
- Do not add another animation or gameplay clock.
- Wrong answers must not restart, shorten, or reposition the active target.
- Preserve the current HorizontalPlayGate and its idle, approach, urgent, correct, tier-up, and miss states.
- Preserve reduced-motion behavior.
- Preserve audio behavior.
- Preserve note, string, fret, score, combo, accuracy, fluency, analytics, run history, cell progress, and best-score behavior where applicable.
